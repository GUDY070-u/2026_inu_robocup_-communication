# SML 경기별 적재·하역 로직

이 문서는 2026-06-29 현재 `sml_system_pkg`의 실제 구현을 기준으로 Production,
Recycling, Lifecycle 경기에서 AMR이 무엇을 어디서 적재하고 어디에 하역하는지
정리한 문서이다.

주요 기준 코드는 다음과 같다.

- 계획 생성: `planning/planner_core.py`
- 슬롯 설정: `planning/planner_config.py`
- 계획 실행 및 ARM 슬롯 변환: `sml_manager_node.py`
- A/B 경기장 ID 변환: `arena_side_utils.py`

## 1. 용어와 공통 원칙

### 1.1 LOAD와 UNLOAD의 기준

`Step.action`의 LOAD와 UNLOAD는 항상 **AMR 기준**이다.

| 동작 | 의미 |
|---|---|
| AMR LOAD | 스테이션 또는 WB에 있는 물체를 AMR 적재 공간으로 싣는다. |
| AMR UNLOAD | AMR 적재 공간의 물체를 스테이션 또는 WB에 내려놓는다. |
| AMR ASSEMBLE | AMR 내부에서 조립한다. |
| WB PRODUCE | 고정 WB 로봇이 조립한다. |
| WB RECYCLE | 고정 WB 로봇이 완성품을 원재료로 분해한다. |
| AMR GOAL | 모든 선행 작업이 끝난 뒤 시작/골 위치로 복귀한다. |

일반 AMR LOAD/UNLOAD 단계의 실행 순서는 다음과 같다.

1. AMR이 `Step.station_id`로 이동한다.
2. 이동이 끝나면 ARM 서비스에 LOAD 또는 UNLOAD를 요청한다.
3. ARM 작업이 성공하면 내비게이터 후처리를 실행한다.
4. 후처리까지 성공하면 해당 Step을 완료 처리한다.

AMR ASSEMBLE만 예외다. 목적지 이동과 AMR 내부 조립을 동시에 시작하고, 두 작업이
모두 끝난 뒤 내비게이터 후처리를 실행한다.

### 1.2 AMR 적재 공간

| 적재 공간 | 용도 |
|---|---|
| 1번 | 완성품, WB 생산 완성품, 기본 recycle 제품 |
| 2~6번 | 원재료 슬라이드 |
| 7~8번 | AMR 조립 공간, 작은 recycle 제품의 보조 선적재 공간 |

원재료 크기와 슬라이드 용량은 다음과 같다.

- 원재료 1~4: 크기 1
- 원재료 5~8: 크기 2
- 원재료 슬라이드 하나의 용량: 3

Production의 AMR 조립에서는 같은 주문의 서로 다른 상부 원재료가 같은 원재료
슬라이드를 사용하지 않는다. 서로 다른 주문은 용량이 허용되는 경우 같은
원재료 슬라이드를 공유할 수 있다.

### 1.3 논리 `slide_id`

일반 적재 위치는 아래 형식으로 기록한다.

```text
slide_id = order_index × 10 + slot_index
```

예를 들어 주문 인덱스 2의 원재료가 4번 적재 공간을 사용하면 `slide_id`는 24다.
물리 적재 공간은 `abs(slide_id) % 10`으로 구한다.

재활용 원재료를 storage로 반환할 때는 음수 ID를 사용한다.

```text
slide_id = -(반환할 로컬 station_id × 10 + slot_index)
```

반환용 음수 ID에는 반환 목적지와 AMR 적재 위치가 함께 들어 있다.

### 1.4 ARM에 전달하는 물리 위치

플래너의 2~6번 논리 슬롯은 ARM 명령 직전에 물리 원재료 슬라이드 1~5번으로
변환된다.

```text
플래너 슬롯 2~6 → 물리 원재료 슬라이드 1~5
ARM slide_id = 물리 슬라이드 번호 × 10 + 적재 위치
```

- 크기 1 원재료의 위치: 0, 2, 4
- 크기 2 원재료의 위치: 1, 3
- 완성품 공간 1번과 조립 공간 7~8번은 별도 변환 없이 물리 슬롯 번호를 전달한다.
- LOAD 때 배정한 세부 위치는 이후 ASSEMBLE과 UNLOAD에서도 동일하게 조회한다.
- UNLOAD 또는 ASSEMBLE이 성공하면 소비된 원재료의 점유 정보를 해제한다.
- AMR 조립 완성품은 첫 번째 입력인 기준 재료의 조립 공간에 남는다.

### 1.5 경기장 스테이션

| 구분 | Side A | Side B |
|---|---:|---:|
| 시작/골 | 0 | 9 |
| Storage shelf | 1, 2, 4, 5 | 10, 11, 13, 14 |
| Workbench | 3, 6, 7 | 12, 15, 16 |
| 실제 사용하는 고정 WB 로봇 | 6 | 16 |
| Customer | 8 | 17 |

플래너 내부 계산은 로컬 ID 0~8을 사용한다. Side B 입력 9~17은 계획 전에
0~8로 변환하고, 생성된 Step은 실행 전에 다시 9~17로 복원한다. Side B의 고정
WB 로봇 16번은 플래너 내부에서 로컬 7번으로 처리한다.

## 2. Production 경기

Production 주문은 제품 종류에 따라 두 경로로 나뉜다.

| 생산 경로 | 제품 ID |
|---|---|
| AMR 내부 조립 | 34, 13, 81, 442, 241, 462, 711, 4482 |
| 고정 WB 조립 | 8518, 48132, 46262 |

### 2.1 AMR 내부 조립 제품

AMR 조립 제품은 최대 두 주문씩 묶어서 준비한다. 조립 공간이 7번과 8번 두
곳이기 때문이다.

전체 흐름은 다음과 같다.

```text
Storage에서 원재료 LOAD
→ AMR 내부 조립 공간/원재료 슬라이드에 적재
→ Customer로 이동하면서 AMR ASSEMBLE
→ Customer에 완성품 UNLOAD
```

#### 적재

1. 각 주문 레시피의 첫 번째 원재료를 기준 재료로 선택한다.
2. 첫 번째 주문의 기준 재료는 조립 공간 7번에 적재한다.
3. 두 번째 주문의 기준 재료는 조립 공간 8번에 적재한다.
4. 나머지 상부 원재료는 2~6번 원재료 슬라이드에 배정한다.
5. 같은 storage에서 가져올 물체는 하나의 LOAD Step으로 묶는다.
6. 여러 storage를 방문해야 하면 이동 비용을 고려해 방문 순서를 정하고,
   앞 LOAD가 끝난 뒤 다음 LOAD가 실행되도록 의존성을 연결한다.

#### 조립과 하역

1. 선택된 배치의 필요한 LOAD가 완료되면 AMR ASSEMBLE을 시작한다.
2. 한 배치에 제품이 두 개 있으면 하나의 ASSEMBLE 명령에 두 제품과 각 입력
   슬롯을 함께 전달한다.
3. AMR은 Customer로 이동하는 동안 배치의 제품을 순서대로 내부 조립한다.
4. 각 조립은 해당 제품 `slide_ids`의 첫 번째 위치, 즉 기준 재료가 놓인 7번
   또는 8번 조립 공간을 결과 위치로 사용한다.
5. 사용한 상부 원재료 슬롯은 비우고, 완성품은 각 기준 재료의 조립 공간에
   남긴다.
6. Customer 도착과 배치 ASSEMBLE이 모두 완료되면 준비된 완성품을 하나의
   UNLOAD Step으로 함께 하역한다.

현재 조기 조립의 단위는 개별 제품이 아니라 최대 두 제품으로 구성된 배치다.
첫 제품의 원재료만 준비된 시점에 다음 storage로 이동하면서 먼저 조립하는
세부 스케줄링은 아직 적용하지 않는다. 현재 구현은 모든 배치 입력을 확보한 뒤
Customer 이동시간과 두 제품의 연속 조립시간을 겹친다.

### 2.2 고정 WB 조립 제품

WB 전용 제품은 모든 원재료를 고정 WB로 운반한 뒤 WB 로봇이 조립한다.

```text
Storage에서 원재료 LOAD
→ 고정 WB에 원재료 UNLOAD
→ WB PRODUCE
→ WB에서 완성품을 AMR 1번 공간에 LOAD
→ Customer에 완성품 UNLOAD
```

#### 원재료 적재와 WB 하역

1. 각 원재료에 2~6번 중 서로 다른 원재료 슬롯을 배정한다.
2. 같은 storage의 원재료는 한 LOAD Step으로 묶는다.
3. 모든 원재료 LOAD가 끝나면 고정 WB로 이동한다.
4. 이전 WB 작업이 남아 있으면 그 작업이 끝날 때까지 기다린다.
5. AMR의 원재료를 WB에 한 번에 UNLOAD한다.

#### 완성품 적재와 Customer 하역

1. 원재료 UNLOAD가 끝나면 WB PRODUCE를 실행한다.
2. WB 조립이 완료되면 완성품을 AMR의 1번 완성품 공간에 LOAD한다.
3. 이 LOAD가 끝나는 시점부터 WB가 다음 작업을 받을 수 있다.
4. 뒤에 AMR 조립 주문이 없으면 AMR은 Customer로 이동하여 1번 공간의
   완성품을 UNLOAD한다.
5. 뒤에 AMR 조립 주문이 있으면 마지막 WB 완성품 한 개는 Customer 하역을
   보류한다. AMR은 이 제품을 1번 공간에 실은 상태로 AMR 조립 원재료를
   수집하고, AMR ASSEMBLE 완료 후 7~8번 완성품과 함께 Customer에 UNLOAD한다.

## 3. Recycling 경기

Recycling은 완성품을 Customer에서 가져와 고정 WB에서 분해한 뒤, 나온
원재료를 지정된 storage shelf로 반환하는 경기다.

여러 recycle 주문이 있을 때는 WB가 쉬는 시간을 줄이기 위해 선적재와 반환을
겹치는 파이프라인을 사용한다.

```text
Customer에서 recycle 제품 선적재
→ 제품을 WB에 UNLOAD
→ WB RECYCLE
→ 분해 원재료를 WB에서 LOAD
→ 다음 제품을 WB에 UNLOAD
→ 다음 WB RECYCLE 동안 이전 원재료를 storage에 UNLOAD
```

### 3.1 Recycle 제품 선적재

기본 제품 적재 위치는 1번이다.

- 원재료 1~4만으로 구성된 작은 제품은 보조 선적재 공간 7~8번도 사용할 수 있다.
- 원재료 5~8이 하나라도 포함된 큰 제품은 1번을 사용한다.
- 최초 선적재 묶음은 최대 두 제품까지만 선택한다.
- 큰 제품과 작은 제품을 함께 싣는 경우 큰 제품은 1번, 작은 제품은 7번 또는
  8번을 사용한다.
- 모든 제품이 작으면 첫 제품은 1번, 추가 제품은 7~8번을 사용한다.
- 실제 WB 처리 순서는 물리 슬롯 번호가 아니라 선택된 주문 순서를 유지한다.

추가 제품 선적재는 다음 조건을 보수적으로 확인한다.

- 사용할 수 있는 보조 공간이 있어야 한다.
- 가능하면 반환 목적 storage가 겹치는 제품을 함께 선택한다.
- WB가 현재 제품을 분해하는 시간 안에 Customer 방문, 제품 LOAD, WB 복귀가
  가능하다고 추정될 때만 추가 선적재한다.

### 3.2 WB 투입과 분해

1. Customer에서 recycle 제품을 AMR에 LOAD한다.
2. 이전 WB 작업이 끝난 뒤 해당 제품을 고정 WB에 UNLOAD한다.
3. 제품 UNLOAD 직후 그 AMR 슬롯은 재사용할 수 있다.
4. 제품이 WB에 들어가면 WB RECYCLE을 시작한다.
5. WB가 분해 중일 때 AMR은 다음 제품 선적재 또는 이전 원재료 반환을 수행할 수
   있지만, WB 자체에는 접근하지 않는다.

### 3.3 분해 원재료 회수

다음 recycle 제품이 대기 중이면 WB 교환 시점의 순서는 다음과 같다.

1. 현재 WB RECYCLE 완료를 기다린다.
2. 분해된 원재료를 WB에서 AMR 2~6번 슬롯으로 LOAD한다.
3. 대기 중인 다음 recycle 제품을 WB에 UNLOAD한다.
4. 다음 WB RECYCLE을 시작한다.
5. 다음 분해가 진행되는 동안 직전 원재료를 storage로 반환한다.

마지막 제품은 WB RECYCLE 완료 후 원재료를 회수하고 모두 반환한 다음 GOAL로
복귀한다.

### 3.4 원재료 반환 적재와 하역

1. 원재료마다 반환 대상 storage를 결정한다.
2. 반환 대상은 초기 task에서 해당 원재료의 반환 자리로 분류된 storage를
   우선 사용한다.
3. 명시된 반환 자리가 없으면 첫 번째 storage를 기본 목적지로 사용한다.
4. 크기 2 원재료부터 먼저 배치하면서 2~6번 슬라이드에 최대한 조밀하게 싣는다.
5. 같은 슬라이드의 총 크기는 3을 넘을 수 없다.
6. 목적지가 같은 원재료는 같은 storage의 UNLOAD Step으로 묶는다.

중간 반환 운행은 다음 원재료를 다시 WB에서 실어야 하므로 WB에서 먼 storage를
먼저 방문하고 WB 가까이에서 끝내는 방향으로 정렬한다. 마지막 반환 운행은
최종 GOAL 복귀가 짧아지도록 GOAL에 가까운 storage를 마지막에 방문한다.

## 4. Lifecycle 경기

Lifecycle은 Production과 Recycling을 함께 수행하면서, recycle에서 나온
원재료를 production에 우선 재사용한다.

핵심은 아래 세 집합이다.

```text
P = 모든 Production 주문에 필요한 원재료
R = 모든 Recycling 제품을 분해했을 때 나오는 원재료
C = P와 R의 중복 원재료(수량까지 고려한 교집합)
```

- `P - C`: 경기 시작 시 storage에서 가져와야 하는 원재료
- `C`: recycle 후 바로 production에 재사용할 원재료
- `R - C`: production에 쓰지 않고 storage로 반환할 원재료

### 4.1 전체 계획 순서

현재 플래너는 다음 순서로 Step을 생성한다.

1. Production에 원재료를 공급하는 연결 recycle 주문
2. WB 전용 Production 주문
3. AMR 조립 가능 Production 주문
4. Production과 연결되지 않은 독립 recycle 주문
5. 연결 recycle에서 남은 원재료 반환
6. 모든 Step 완료 후 GOAL 복귀

Step 생성 순서와 실제 실행 시점은 완전히 같지는 않다. 서로 독립적인 AMR/WB
Step은 `depends_on` 조건과 AMR/WB busy 상태가 허용하면 병렬로 실행될 수 있다.

### 4.2 Production과 연결된 recycle

재사용할 원재료를 만드는 recycle 제품은 Production보다 먼저 처리한다.

```text
Customer에서 recycle 제품 LOAD
→ 고정 WB에 제품 UNLOAD
→ WB RECYCLE
→ 재사용 원재료는 WB에 대기
→ 남는 원재료는 나중에 storage로 반환
```

제품 운반에는 1번 또는 작은 제품용 7~8번 공간을 사용한다. WB RECYCLE이 끝난
Step ID는 해당 원재료를 사용할 Production Step의 의존성으로 연결된다.

### 4.3 재활용 원재료를 WB 전용 생산에 사용하는 경우

WB 전용 제품이 재활용 원재료를 사용하면 해당 원재료는 이미 WB에 있으므로
AMR이 다시 LOAD/UNLOAD하지 않는다.

1. storage에서 필요한 나머지 원재료만 AMR에 LOAD한다.
2. recycle 완료와 나머지 원재료의 WB UNLOAD를 모두 기다린다.
3. WB PRODUCE를 실행한다.
4. 완성품을 AMR 1번 공간에 LOAD하여 Customer에 UNLOAD한다.

### 4.4 재활용 원재료를 AMR 조립 생산에 사용하는 경우

AMR 조립 제품이 재활용 원재료를 사용하면 원재료가 WB에 있으므로 AMR이 WB에서
해당 원재료를 LOAD한다.

1. recycle 완료를 기다린다.
2. 기준 재료라면 WB에서 조립 공간 7번 또는 8번으로 LOAD한다.
3. 상부 재료라면 WB에서 원재료 공간 2~6번으로 LOAD한다.
4. storage에서 가져오는 다른 원재료와 합쳐 AMR ASSEMBLE을 수행한다.
5. Customer에 완성품을 UNLOAD한다.

WB에서 재활용 원재료를 싣는 LOAD는 WB가 다른 작업 중일 때 실행되지 않도록
`last_wb_clear_step_id` 의존성이 추가된다.

### 4.5 남은 원재료 반환

연결 recycle에서 생산에 사용하지 않은 `R - C` 원재료는 즉시 반환하지 않고
deferred 목록에 모은다.

1. 관련 WB RECYCLE이 모두 끝난 뒤 원재료를 WB에서 LOAD한다.
2. 2~6번 슬롯의 용량을 기준으로 가능한 적은 운행 수로 나눈다.
3. 각 운행에서 목적 storage별로 묶어 UNLOAD한다.
4. 중간 운행은 WB 복귀가 짧도록, 마지막 운행은 GOAL 복귀가 짧도록 순서를
   정한다.

Production과 연결되지 않은 recycle 주문은 3장의 일반 Recycling 파이프라인으로
처리한다.

## 5. 의존성과 충돌 방지 규칙

- 하나의 AMR Step이 실행 중이면 다른 AMR Step은 시작하지 않는다.
- 하나의 WB Step이 실행 중이면 다른 WB Step은 시작하지 않는다.
- AMR 트랙과 WB 트랙은 의존성이 없으면 병렬 실행할 수 있다.
- AMR이 고정 WB에서 LOAD 또는 UNLOAD할 때는 진행 중인 WB 작업의 완료를
  기다린다.
- AMR이 WB행 LOAD/UNLOAD를 선택하면 출발 전에 WB 사용권을 예약한다.
- AMR이 WB 사용권을 예약한 동안에는 새로운 WB PRODUCE/RECYCLE을 시작하지
  않는다.
- WB가 작업 중이면 WB 접근 Step은 보류하고, 실행 가능한 다른 AMR Step을
  먼저 수행한다.
- WB 조립/분해는 필요한 물체가 WB에 UNLOAD된 뒤 시작한다.
- 같은 물리 제품 슬롯 1/7/8은 앞 제품이 WB에 UNLOAD되기 전에 재사용하지 않는다.
- 반환 운행 중에는 다음 WB 원재료 LOAD를 시작하지 않는다.
- 최종 GOAL Step은 생성된 모든 앞 Step에 의존한다.

## 6. 경기별 요약

| 경기 | 적재 출발지 | 주요 AMR 적재 공간 | 하역 목적지 | 핵심 최적화 |
|---|---|---|---|---|
| Production-AMR | Storage 또는 Lifecycle의 WB | 기준 재료 7~8, 상부 재료 2~6 | Customer | 최대 2개 준비, 이동 중 조립 |
| Production-WB | Storage 또는 Lifecycle의 WB | 원재료 2~6, 완성품 1 | WB 후 Customer | WB 작업과 독립 AMR 작업 병렬화 |
| Recycling | Customer, 이후 WB | 제품 1/7/8, 반환 원재료 2~6 | WB 후 Storage | 다음 제품 선적재와 이전 원재료 반환 교차 |
| Lifecycle | Storage, Customer, WB | 위 세 경로를 모두 사용 | WB, Customer, Storage | recycle 원재료를 production에 우선 재사용 |

## 7. 구현상 주의사항

- `order_index`는 0부터 시작한다.
- batch ID 10, 20, ..., 80은 각각 원재료 1, 2, ..., 8의 5개 묶음으로
  해석하지만 ARM 명령에는 실제 원재료 ID 1~8을 전달한다.
- 현재 WB 전용 제품은 8518, 48132, 46262로 고정되어 있다.
- 현재 원재료 크기 규칙, 슬롯 용량, 제품 레시피는
  `planning/planner_config.py`와 `_raw_size()` 구현에 의존한다.
- 스테이션·슬롯·제품 분류가 바뀌면 이 문서와 함께 설정 및 회귀 테스트를
  갱신해야 한다.
